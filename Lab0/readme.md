# Перемножение матриц: GPU vs CPU (CUDA)

В данном проекте сравнивается производительность умножения квадратных матриц на **CPU (один поток)** и **GPU (с использованием CUDA / PyCUDA)**.

## Результаты измерений

| Размерность | GPU (сек) | CPU (сек) | Ускорение |
| :---------: | :-------: | :-------: | :-------: |
|   50x50     | 0.001001  | 0.065030  |  ~65x     |
|  100x100    | 0.000998  | 0.503017  | ~504x     |
|  200x200    | 0.001999  | 4.070234  | ~2036x    |
|  400x400    | 0.005035  | 32.141670 | ~6384x    |
|  800x800    | 0.069134  | 270.341954| ~3910x    |
| 1600x1600   | 0.164995  |    —      |    —      |
| 3200x3200   | 0.666998  |    —      |    —      |
| 6400x6400   | 3.898160  |    —      |    —      |
|12800x12800  | 31.147587 |    —      |    —      |

> Для больших размеров на CPU было затрачено очень много времени

## Характеристики системы

Тестирование проводилось на следующем оборудовании:

- **CPU:** Intel Xeon E5   
- **GPU:** NVIDIA T4
- **Технологии:** CUDA, PyCUDA, NumPy  

---

## Анализ результатов

### Малые размеры (до 100x100)

Для небольших матриц GPU уже показывает преимущество, но оно не критично с практической точки зрения:

- накладные расходы на запуск CUDA частично нивелируют выигрыш
- CPU справляется достаточно быстро

Вывод: для очень маленьких задач использование GPU не всегда оправдано.

---

### Средние размеры (200x200 — 800x800)

Здесь GPU начинает раскрывать свой потенциал:

- ускорение достигает тысяч раз
- CPU время растёт квадратично/кубически из-за тройного цикла

Вывод: начиная со средних размеров, GPU становится значительно эффективнее.

---

### Большие размеры (1600x1600 и выше)

- CPU становится практически неприменим (в разумное время)
- GPU продолжает масштабироваться и выполнять вычисления за секунды

Пример:
- **6400x6400 → ~3.9 сек на GPU**
- **12800x12800 → ~31 сек на GPU**

Вывод: для больших матриц GPU — практически единственный реалистичный вариант.

---

## Итоговые выводы

- GPU обеспечивает колоссальное ускорение при росте размерности матриц  
- CPU подходит только для:
  - маленьких задач
  - отладки и проверки  
- Основной выигрыш GPU достигается за счёт:
  - массивного параллелизма
  - эффективной работы с большими объёмами данных  

---

## Заключение

Данный эксперимент наглядно показывает, что GPU — это ключевой инструмент для задач линейной алгебры и высокопроизводительных вычислений, особенно при работе с большими матрицами.
