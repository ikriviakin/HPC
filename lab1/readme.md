## Характеристики ПК

Процессор: Intel(R) Xeon E5
Видеокарта: NVIDIA T4 Gpu

## Итог

1. **Малые объемы данных (2000–8000)**

   - На небольших размерах CPU либо быстрее, либо сопоставим с GPU.
   - Основная причина — накладные расходы на запуск CUDA (копирование памяти + запуск ядра).

2. **Средние объемы данных (16000–128000)**

   - GPU начинает стабильно выигрывать.
   - Параллелизм начинает окупать накладные расходы.

3. **Большие объемы данных (256000 и выше)**

   - GPU значительно быстрее CPU.
   - Разница становится кратной:
     - 1M элементов: ~0.10 сек (CPU) vs ~0.002 сек (GPU)
     - 8M элементов: ~0.80 сек (CPU) vs ~0.009 сек (GPU)

## Заключение

GPU показывает явное преимущество при работе с большими массивами данных за счёт массового параллелизма.

Однако на малых объемах данных использование GPU неэффективно из-за накладных расходов на:
- копирование данных между CPU и GPU
- запуск CUDA-ядра

Оптимальная зона применения GPU — задачи с большим объемом данных и высокой степенью параллелизма.
